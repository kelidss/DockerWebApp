Introdução ao Docker

Docker é uma ferramenta que simplifica a criação e execução de aplicativos ao colocá-los em containers. Pense em um container como uma caixa que contém tudo o que um aplicativo precisa para funcionar, incluindo o próprio aplicativo, suas dependências e configurações. Isso garante que o aplicativo funcione da mesma forma em qualquer lugar, seja no seu computador, no de um colega ou na nuvem.

Essa abordagem facilita o compartilhamento e a portabilidade dos aplicativos, pois qualquer pessoa pode rodar o mesmo container e obter os mesmos resultados. Com Docker, você evita problemas de compatibilidade e configuração, tornando o desenvolvimento e a execução de aplicativos mais rápidos e confiáveis.

Instalação

Windows

Para instalar o Docker no Windows, siga estes passos:

  1. Acesse a página de download do Docker Desktop.
  2. Baixe o instalador para Windows.
  3. Execute o instalador e siga as instruções na tela.
  4. Após a instalação, inicie o Docker Desktop e siga as instruções de configuração.

Mac

Para instalar o Docker no Mac, siga estes passos:

  1. Acesse a página de download do Docker Desktop.
  2. Baixe o instalador para Mac.
  3. Abra o arquivo .dmg e arraste o ícone do Docker para a pasta de Aplicativos.
  4. Inicie o Docker Desktop a partir da pasta de Aplicativos e siga as instruções de configuração.

Linux

Para instalar o Docker no Linux, siga estes passos:

  1. Abra um terminal.
  2. Atualize o índice de pacotes com o comando:
    3. sudo apt-get update
  3. Instale o Docker com o comando:
    4. sudo apt-get install docker-ce
  4. Verifique se o Docker foi instalado corretamente com o comando:
    5. docker --version
  5. Para permitir que o Docker seja executado sem sudo, adicione seu usuário ao grupo docker:
    6. sudo usermod -aG docker $USER
  6. Reinicie sua sessão para aplicar as mudanças.

Comandos Básicos

  • docker pull <imagem>: Baixa uma imagem do Docker Hub ou de um registro. Exemplo: docker pull nginx
  • docker run <opções> <imagem>: Executa um novo container a partir de uma imagem. Exemplo: docker run -d -p 8080:80 nginx
  • docker ps: Lista os containers em execução. Use -a para listar todos os containers, incluindo os parados.
  • docker stop <container_id>: Para um container em execução. Exemplo: docker stop meu-container
  • docker start <container_id>: Inicia um container parado. Exemplo: docker start meu-container
  • docker restart <container_id>: Reinicia um container em execução. Exemplo: docker restart meu-container
  • docker rm <container_id>: Remove um container. Exemplo: docker rm meu-container
  • docker rmi <imagem>: Remove uma imagem do Docker. Exemplo: docker rmi nginx
  • docker logs <container_id>: Exibe os logs de um container. Exemplo: docker logs meu-container
  • docker exec -it <container_id> /bin/bash: Executa um comando dentro de um container em execução, geralmente usado para acessar um terminal interativo. Exemplo: docker exec -it meu-container /bin/bash
  • docker-compose up: Inicia os serviços definidos em um arquivo docker-compose.yml. Use -d para rodar em segundo plano.
  • docker-compose down: Para e remove os containers, redes e volumes definidos no arquivo docker-compose.yml.

Docker Compose

O Docker Compose é uma ferramenta que permite definir e gerenciar aplicações multi-container com Docker. Em vez de iniciar containers individualmente, você pode usar um arquivo de configuração chamado docker-compose.yml para descrever todos os serviços, redes e volumes necessários para sua aplicação.

Com o Docker Compose, você pode:

  • Definir Serviços: Especificar quais containers serão usados e como eles se comunicam.
  • Gerenciar Dependências: Definir dependências entre serviços, como bancos de dados e aplicativos web.
  • Executar Todos os Serviços: Usar um único comando para iniciar todos os serviços descritos no arquivo de configuração.
  • Facilitar o Desenvolvimento: Configurar ambientes de desenvolvimento consistentes e reproduzíveis.

Exemplo de um arquivo docker-compose.yml

version: '3'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  db:
    image: postgres
    environment:
      POSTGRES_DB: mydatabase
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:

Dicas Finais

Docker é uma ferramenta poderosa que pode transformar a forma como você desenvolve, testa e implanta suas aplicações. Aqui estão algumas dicas finais para aproveitar ao máximo o Docker:

  • Documente suas Imagens e Containers: Inclua informações sobre como usar e configurar suas imagens e containers em seus arquivos README.
  • Mantenha suas Imagens Atualizadas: Regularmente atualize suas imagens base e recompile seus containers para garantir que você tenha as últimas atualizações de segurança.
  • Utilize Volumes para Persistência: Use volumes para armazenar dados persistentes fora do container, evitando a perda de dados ao remover containers.
  • Teste Seus Containers: Realize testes abrangentes para garantir que seus containers funcionem como esperado em diferentes ambientes.
  • Monitore o Desempenho: Use ferramentas de monitoramento para acompanhar o desempenho e o uso de recursos de seus containers.

Exemplo Prático: Aplicação com Flask e PostgreSQL

Vamos criar um exemplo prático de uma aplicação que utiliza Flask como framework web e PostgreSQL como banco de dados, utilizando Docker para gerenciar o ambiente.

Passo 1: Criar um arquivo Dockerfile

FROM python:3.9-slim
    
    WORKDIR /app
    
    COPY requirements.txt requirements.txt
    RUN pip install -r requirements.txt
    
    COPY . .
    
    CMD ["flask", "run", "--host=0.0.0.0"]

Passo 2: Criar um arquivo docker-compose.yml

version: '3'
    services:
      web:
        build: .
        ports:
          - "5000:5000"
        environment:
          - FLASK_APP=app.py
          - FLASK_ENV=development
        depends_on:
          - db
    
      db:
        image: postgres:13
        environment:
          POSTGRES_DB: mydatabase
          POSTGRES_USER: user
          POSTGRES_PASSWORD: password
        volumes:
          - db_data:/var/lib/postgresql/data
    
    volumes:
      db_data:

Passo 3: Criar o arquivo requirements.txt

Flask
    psycopg2-binary

Passo 4: Criar o arquivo app.py

from flask import Flask
    import psycopg2
    
    app = Flask(__name__)
    
    @app.route('/')
    def hello_world():
        return 'Hello, Docker!'
    
    if __name__ == '__main__':
        app.run(debug=True)

Passo 5: Rodar o projeto

docker-compose up --build

Com esses passos, você terá uma aplicação Flask rodando em um container Docker e um banco de dados PostgreSQL em outro container, todos gerenciados pelo Docker Compose.
